# Android Platform Plan — HAM HAT Control Center v4

Status date: 2026-04-05
Target: `app/android/`

## Goals

Bring HAM HAT Control Center to Android phones and tablets, maximising
reuse of the existing Python engine while providing a native-feeling
Material Design UI.

Primary use-cases:
1. **PAKT BLE mode** — phone as wireless radio controller via BLE
2. **SA818 via USB OTG** — phone/tablet driving SA818 over USB serial adapter
3. **APRS receive/transmit** — using the phone microphone / audio jack
4. **Profile management** — import/export profiles to/from the desktop app

## Architecture

### UI Framework: Kivy + KivyMD

| Option | Pros | Cons |
|--------|------|------|
| Kivy + KivyMD | Python reuse; one codebase | Larger APK; non-native feel |
| React Native / Flutter | Native feel | Full rewrite; no Python reuse |
| Native Kotlin | Best UX / APIs | Full rewrite; no Python reuse |
| PWA | Fast to build | No USB serial; limited BLE |

Decision: **Kivy 2.3 + KivyMD 1.2** via Buildozer.

- Python-first: reuses all `app/engine/` non-UI code as-is
- KivyMD gives Material Design look that feels modern on Android
- Buildozer packages into a signed APK for phone/tablet

### Platform Adapters (`app/android/hal/`)

> **Note:** Named `hal/` (Hardware Abstraction Layer) rather than `platform/` to avoid colliding
> with Python's stdlib `platform` module — an import that `app.engine.platform_paths` relies on.

| File | Purpose |
|------|---------|
| `paths.py` | Android data/log dirs using `kivy.app.App.user_data_dir` |
| `ble_manager.py` | BLE scan/connect via bleak with Android backend; background asyncio thread |
| `serial_manager.py` | USB serial via `usbserial4a` (pyjnius-backed) on Android; pyserial on desktop |
| `audio_manager.py` | Android audio via jnius AudioTrack; delegates to engine audio_tools on desktop |

## Directory Layout

```
app/android/
├── main.py                    # KivyMD app entry point; responsive phone/tablet layout
├── engine_bridge.py           # sys.path setup to import app.engine.*

│   └── audio_manager.py       # Audio enumeration; play_tone(); jnius on Android
├── screens/
│   ├── __init__.py
│   ├── control_screen.py      # SA818/DigiRig/PAKT panels; BLE state chip; audio routing
```

## Build Requirements

Build machine must be Linux or macOS with:
- Python 3.10+
- Buildozer 1.5+: `pip install buildozer`
- Android SDK + NDK (Buildozer downloads automatically)
- Java JDK 17
- `apt install -y git zip unzip openjdk-17-jdk autoconf libtool pkg-config zlib1g-dev libncurses5-dev libncursesw5-dev libtinfo5 cmake libffi-dev libssl-dev`

Build command:
```bash
cd app/android
buildozer android debug       # debug APK
buildozer android release     # release APK (needs signing key)
buildozer android debug deploy run  # deploy to connected device
```

APK output: `app/android/bin/HamHatCC-4.0-arm64-v8a-debug.apk`

## Open Risks

| Risk | Severity | Note |
|---|---|---|
| bleak Android backend stability | Medium | Works in 0.22+; test with actual BLE PAKT device |
| sounddevice on Android | Medium | May need pyjnius AudioRecord fallback |
| USB serial OTG | Medium | Needs `usbserial4a`; device must support USB host mode |
| APK size | Low | Kivy+KivyMD+Python runtime ≈ 60–90 MB |
| Google Play distribution | Low | BLE + location permissions may trigger review |

## Validation Checklist (per build)

1. APK builds without errors: `buildozer android debug`
2. Installs on Android 8+ (API 26+): `adb install bin/*.apk`
3. App launches without crash (2+ seconds)
4. Bottom navigation switches screens (phone)
5. Navigation rail works on tablet (≥ 600 dp)
6. Profile loads from disk on launch
7. Profile saves on app exit / explicit save
8. BLE scan finds PAKT device (needs hardware)
9. BLE connect succeeds and shows CONNECTED state
10. SA818 USB serial connects (needs USB OTG + SA818)
